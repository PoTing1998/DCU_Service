# UITest 简化版消息发送功能 - 使用说明

## 概述

为了方便快速测试，提供了简化版的消息发送功能，只需添加**5个控件**即可完成基本测试。

## 界面布局（简化版）

### 在现有的 TabPage 中添加以下控件：

建议在**"资料庫設定"** TabPage（tabDatabaseSettings）底部添加，或者创建新的 TabPage。

---

### 需要添加的控件（共5个）

#### 1. 消息内容输入框
- **名称**: `txtSimpleMessageText`
- **类型**: TextBox
- **属性**:
  - Multiline = true
  - Size = 400 x 60
  - Text = "UITest 测试消息 - Test Message"

#### 2. 目标设备输入框
- **名称**: `txtSimpleTargetDevice`
- **类型**: TextBox
- **属性**:
  - Size = 200 x 20
  - Text = "LG01_CCS_CDU-1"

#### 3. 即时消息复选框
- **名称**: `chkSimpleInstantMessage`
- **类型**: CheckBox
- **属性**:
  - Text = "即时消息（否则为预录消息）"
  - Checked = false

#### 4. 发送按钮
- **名称**: `btnSimpleSendMessage`
- **类型**: Button
- **属性**:
  - Text = "发送测试消息"
  - Size = 120 x 40
  - Click 事件: `btnSimpleSendMessage_Click`

#### 5. 结果显示文本框
- **名称**: `txtSimpleResult`
- **类型**: TextBox
- **属性**:
  - Multiline = true
  - ScrollBars = Vertical
  - ReadOnly = true
  - Size = 600 x 300
  - Dock = Bottom（可选）

---

### 可选控件

#### 6. 清除结果按钮
- **名称**: `btnSimpleClearResult`
- **类型**: Button
- **属性**:
  - Text = "清除结果"
  - Click 事件: `btnSimpleClearResult_Click`

#### 7. 使用默认值按钮
- **名称**: `btnSimpleUseDefaults`
- **类型**: Button
- **属性**:
  - Text = "填充默认值"
  - Click 事件: `btnSimpleUseDefaults_Click`

---

## 快速添加步骤（Visual Studio）

### 方法 1: 在现有 TabPage 中添加

1. 打开 Visual Studio
2. 打开 `UITest` 项目
3. 双击 `Form1.cs` 打开设计器
4. 选择 **"资料庫設定"** TabPage（tabDatabaseSettings）
5. 从工具箱拖拽以下控件到 TabPage 底部：
   - 2 个 Label（用于提示）
   - 3 个 TextBox
   - 1 个 CheckBox
   - 3 个 Button
6. 设置各控件的 Name 属性（见上方列表）
7. 双击"发送测试消息"按钮，自动生成事件处理程序
8. Visual Studio 会自动链接到 `Form1.MessageSenderSimple.cs` 中的方法

### 方法 2: 手动编辑 Designer.cs（仅当必要时）

在 `Form1.Designer.cs` 的 `tabDatabaseSettings` 控件初始化部分添加新控件。

## 使用流程

### 1. 准备工作

- 确保数据库连接正常（在"资料庫設定"页面测试连接）
- 确保 DCU Service 正在运行（可选，MSMQ 通知需要）

### 2. 发送测试消息

1. 点击"填充默认值"按钮（可选）
2. 修改消息内容（如需要）
3. 确认目标设备格式正确（例如：`LG01_CCS_CDU-1`）
4. 选择消息类型（预录消息或即时消息）
5. 点击"发送测试消息"
6. 查看结果显示

### 3. 查看结果

结果文本框会显示：
- [1/5] 生成消息 ID
- [2/5] 解析目标设备
- [3/5] 插入消息到数据库
- [4/5] 更新播放列表
- [5/5] 发送 MSMQ 通知

如果所有步骤都显示 ✓，则消息发送成功。

## 功能说明

### 自动使用的默认值

简化版会自动使用以下默认参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 消息类型 | 71 | FullWindow（全窗口显示） |
| 消息优先级 | 1 | 最高优先级 |
| 显示模式 | 100 (0x64) | 静态显示（不滚动） |
| 滚动速度 | 5 | 如果滚动时的速度 |
| 播放间隔 | 0 | 无间隔 |
| 字体类型 | 明體 | 0x01 |
| 字体大小 | 24 | 24x24 像素 |
| 字体颜色 | 白色 | RGB(255,255,255) |

### 消息类型区别

- **预录消息**: 插入到 `dmd_pre_record_message` 表，需要消息名称
- **即时消息**: 插入到 `dmd_instant_message` 表，不需要消息名称

两种类型都会：
1. 插入消息到对应的表
2. 更新 `dmd_playlist` 播放列表
3. 通过 MSMQ 通知对应的 Task

## 数据库操作说明

### 执行的数据库操作

1. **插入消息**:
   ```sql
   -- 预录消息
   INSERT INTO dbo.dmd_pre_record_message (...) VALUES (...)

   -- 即时消息
   INSERT INTO dbo.dmd_instant_message (...) VALUES (...)
   ```

2. **更新播放列表**:
   ```sql
   -- 删除旧记录
   DELETE FROM dbo.dmd_playlist
   WHERE station_id = '...' AND area_id = '...' AND device_id = '...'

   -- 插入新记录
   INSERT INTO dbo.dmd_playlist (...) VALUES (...)
   ```

### 查看数据库结果

可以在 PostgreSQL 中执行以下查询验证：

```sql
-- 查看刚插入的预录消息
SELECT * FROM dbo.dmd_pre_record_message
ORDER BY ins_time DESC LIMIT 5;

-- 查看刚插入的即时消息
SELECT * FROM dbo.dmd_instant_message
ORDER BY ins_time DESC LIMIT 5;

-- 查看播放列表
SELECT * FROM dbo.dmd_playlist
WHERE device_id = 'CDU-1'
ORDER BY ins_time DESC LIMIT 5;
```

## MSMQ 通知说明

### MSMQ 消息格式

```
2##<随机ID>##<JSON数据>
```

示例：
```
2##12345##{"JsonObjectName":"ASI.Wanda.DMD.JsonObject.DCU.FromDMD.SendPreRecordMessage","seatID":"OCC01","msg_id":[],"target_du":["LG01_CCS_CDU-1"],"dbName1":"dmd_pre_record_message","dbName2":"dmd_playlist"}
```

### 目标队列路径

根据设备ID自动判断：
- **CDU** → `.\\Private$\\TaskCDU`
- **SDU** → `.\\Private$\\TaskSDU`
- **PUP** → `.\\Private$\\TaskPUP`
- **PDN** → `.\\Private$\\TaskPDN`

### MSMQ 失败处理

如果 MSMQ 发送失败（例如 DCU Service 未运行），不会影响数据库操作。消息已经成功插入数据库和播放列表，可以：

1. 启动 DCU Service
2. 手动触发 Task 重新读取数据库
3. 或者重新点击"发送测试消息"（会更新数据库并重试 MSMQ）

## 故障排查

### 问题 1: 数据库连接失败

**错误信息**: "✗ 发送失败: could not connect to server"

**解决方法**:
1. 检查数据库连接设置（"资料庫設定"页面）
2. 点击"測試連線"按钮验证连接
3. 确认 PostgreSQL 服务正在运行
4. 检查防火墙设置

### 问题 2: MSMQ 发送失败

**错误信息**: "⚠ MSMQ 发送失败: The queue does not exist..."

**原因**: DCU Service 未运行或队列未创建

**解决方法**:
1. 启动 DCU Service (KernelService)
2. 确认 Task 进程正在运行
3. 检查 MSMQ 服务是否启用
4. 查看 Windows 服务管理器

**注意**: MSMQ 失败不影响数据库操作，消息已成功保存

### 问题 3: 目标设备格式错误

**错误信息**: "✗ 错误: 目标设备格式错误"

**解决方法**:
- 确保格式为: `StationID_AreaID_DeviceID`
- 例如: `LG01_CCS_CDU-1`
- 不要有多余的空格
- 大小写需要匹配数据库中的记录

### 问题 4: 显示器没有收到消息

**可能原因**:
1. DCU Service 未运行
2. 串口未连接或配置错误
3. 目标设备ID不匹配
4. Task 处理出错

**排查步骤**:
1. 检查 DCU Service 日志文件
2. 验证数据库中的数据是否正确
3. 确认设备ID与实际硬件匹配
4. 检查串口连接状态

## 完整流程图

```
UITest 界面输入
    ↓
验证输入参数
    ↓
生成 GUID (messageId, playlistId)
    ↓
解析目标设备 (station_id, area_id, device_id)
    ↓
插入消息到数据库
    ├─ dmd_pre_record_message (预录)
    └─ dmd_instant_message (即时)
    ↓
更新播放列表
    ├─ DELETE 旧记录
    └─ INSERT 新记录到 dmd_playlist
    ↓
发送 MSMQ 通知到 Task
    ├─ 判断设备类型 (CDU/SDU/PUP/PDN)
    └─ 发送 JSON 消息到对应队列
    ↓
TaskCDU/SDU/PUP/PDN 接收
    ↓
从数据库读取消息内容
    ├─ 查询 dmd_playlist
    └─ 查询 dmd_*_message
    ↓
组装显示封包
    ├─ 创建 FullWindow
    ├─ 创建 Sequence
    └─ 创建 Packet
    ↓
序列化为 byte[]
    ├─ 转换为 BIG5 编码
    └─ 计算 LRC 校验和
    ↓
通过串口发送
    ↓
显示器接收并显示
```
