## 实现步骤

### 方法 1: 使用 Visual Studio 设计器（推荐）

1. 打开 Visual Studio
2. 打开 `UITest` 项目
3. 双击 `Form1.cs` 打开设计器
4. 在 `tabControl1` 上右键 → "添加选项卡"
5. 设置新 TabPage 的 Name = "tabMessageSender"，Text = "消息发送测试"
6. 从工具箱拖拽控件到新 TabPage 上
7. 按照上述布局设置各个控件的属性
8. 双击按钮添加事件处理程序（已在 Form1.MessageSender.cs 中实现）

### 方法 2: 手动编辑 Designer.cs（高级）

由于 Designer.cs 文件自动生成，建议使用 Visual Studio 设计器来添加控件。如果必须手动编辑，请参考现有 TabPage 的代码结构。

## 使用流程

1. 选择消息类型（预录消息或即时消息）
2. 输入消息内容（中文、英文）
3. 设置字体和显示参数
4. 选择或输入目标设备
5. 点击"发送测试消息"
6. 查看执行结果

---

## 数据库联动流程

```
用户输入消息
    ↓
插入到 dmd_pre_record_message 或 dmd_instant_message
    ↓
更新 dmd_playlist（删除旧记录，插入新记录）
    ↓
发送 MSMQ 消息到 TaskCDU/TaskSDU/TaskPUP/TaskPDN
    ↓
Task 从数据库读取消息内容
    ↓
组装封包并通过串口发送到显示器
```

---

## 注意事项

1. **DCU Service 必须运行**: MSMQ 通知才能被接收
2. **数据库连接**: 确保在"资料庫設定"页面配置正确的数据库连接
3. **设备格式**: 目标设备格式必须为 `StationID_AreaID_DeviceID`（例如：LG01_CCS_CDU-1）
4. **颜色名称**: 使用 ColorHelper 支持的颜色名称（参考 Display/ColorHelper.cs）
