# UITest 与 DCU Service 数据库联动功能 - 实现总结

## 项目概述

成功实现了 **UITest 工具与 DCU Service 的数据库间接联动功能**，允许通过 UITest 界面发送测试消息，经由数据库和 MSMQ 触发 DCU Service 处理，最终显示到硬件设备上。

### ✅ 3. 数据流程

```
UITest 界面
    ↓
[1] 验证输入
    ↓
[2] 生成 GUID
    ↓
[3] 插入消息到数据库
    ├─ dmd_pre_record_message (预录)
    └─ dmd_instant_message (即时)
    ↓
[4] 更新播放列表
    └─ dmd_playlist
    ↓
[5] 发送 MSMQ 通知
    └─ TaskCDU/SDU/PUP/PDN
    ↓
DCU Service 接收
    ↓
从数据库读取消息
    ↓
组装封包
    ↓
串口发送
    ↓
显示器显示
```

## 故障排查

### 问题 1: 编译错误 - 找不到 Newtonsoft.Json

**错误**: `The type or namespace name 'Newtonsoft' could not be found`

**解决**:
- 确认 UITest.csproj 已包含 Newtonsoft.Json 引用
- 检查 DLL 路径: `D:\ASI.Wanda.DMD.DCUService\Newtonsoft.Json.dll`
- 如果路径不存在，修改 UITest.csproj 中的 HintPath

### 问题 2: 编译错误 - 找不到 System.Messaging

**错误**: `The type or namespace name 'Messaging' does not exist in the namespace 'System'`

**解决**:
- 确认 UITest.csproj 已包含 `<Reference Include="System.Messaging" />`
- 在 Visual Studio 中: 项目 → 添加引用 → System.Messaging

### 问题 3: 运行时错误 - 数据库连接失败

**错误**: "could not connect to server"

**解决**:
1. 检查 PostgreSQL 服务是否运行
2. 验证数据库连接信息（IP、端口、用户名、密码）
3. 检查防火墙设置
4. 在"资料庫設定"页面测试连接

### 问题 4: MSMQ 发送失败

**错误**: "The queue does not exist"

**解决**:
1. 启动 DCU Service (KernelService)
2. 确认 MSMQ 服务已启用
3. 检查 Task 进程是否运行
4. 查看 Windows 服务管理器

**注意**: MSMQ 失败不影响数据库操作，消息已成功保存

## 技术细节

### 数据库表结构

#### dmd_pre_record_message
```sql
message_id          UUID PRIMARY KEY
message_name        VARCHAR
message_type        INTEGER  -- 71 = FullWindow
message_priority    INTEGER  -- 1-4
move_mode           INTEGER  -- 100 = 静态
move_speed          INTEGER  -- 滚动速度
interval            VARCHAR
message_content     VARCHAR  -- 中文内容
font_type           VARCHAR  -- 明體
font_size           INTEGER  -- 24
font_color          VARCHAR  -- 白色
message_content_en  VARCHAR  -- 英文内容
font_type_en        VARCHAR
font_size_en        INTEGER
font_color_en       VARCHAR
ins_user, ins_time, upd_user, upd_time
```

#### dmd_instant_message
与 dmd_pre_record_message 相似，但没有 message_name 字段

#### dmd_playlist
```sql
playlist_id     UUID PRIMARY KEY
station_id      VARCHAR
area_id         VARCHAR
device_id       VARCHAR
message_id      UUID  -- 关联到消息表
message_type    INTEGER
send_time       VARCHAR
```

### MSMQ 消息格式

```
<MessageType>##<MessageID>##<JsonData>
```

**示例**:
```
2##12345##{"JsonObjectName":"ASI.Wanda.DMD.JsonObject.DCU.FromDMD.SendPreRecordMessage","seatID":"OCC01","msg_id":[],"target_du":["LG01_CCS_CDU-1"],"dbName1":"dmd_pre_record_message","dbName2":"dmd_playlist"}
```

**字段说明**:
- `MessageType`: 2 = Command
- `MessageID`: 随机数 1-65535
- `JsonData`: JSON 格式的消息数据

### 队列路径映射

| 设备类型 | 队列路径 |
|---------|---------|
| *CDU* | `.\\Private$\\TaskCDU` |
| *SDU* | `.\\Private$\\TaskSDU` |
| *PUP* | `.\\Private$\\TaskPUP` |
| *PDN* | `.\\Private$\\TaskPDN` |
