## 🔄 数据流程

```
┌──────────────┐
│ UITest 界面  │ 用户输入消息内容
└──────┬───────┘
       │
       ↓
┌──────────────────────────────┐
│ [1/5] 生成 GUID              │
│       - 消息 ID              │
│       - 播放列表 ID          │
└──────┬───────────────────────┘
       │
       ↓
┌──────────────────────────────┐
│ [2/5] 解析目标设备           │
│       - Station ID           │
│       - Area ID              │
│       - Device ID            │
└──────┬───────────────────────┘
       │
       ↓
┌──────────────────────────────┐
│ [3/5] 插入消息到数据库       │
│       ├─ dmd_pre_record_msg  │ (预录)
│       └─ dmd_instant_message │ (即时)
└──────┬───────────────────────┘
       │
       ↓
┌──────────────────────────────┐
│ [4/5] 更新播放列表           │
│       - DELETE 旧记录        │
│       - INSERT 新记录        │
│       - dmd_playlist         │
└──────┬───────────────────────┘
       │
       ↓
┌──────────────────────────────┐
│ [5/5] 发送 MSMQ 通知         │
│       ├─ TaskCDU (CDU设备)   │
│       ├─ TaskSDU (SDU设备)   │
│       ├─ TaskPUP (PUP设备)   │
│       └─ TaskPDN (PDN设备)   │
└──────┬───────────────────────┘
       │
       ↓
┌──────────────────────────────┐
│ DCU Service 接收             │
│ - 从数据库读取消息           │
│ - 组装显示封包               │
│ - 通过串口发送               │
└──────┬───────────────────────┘
       │
       ↓
┌──────────────┐
│ 显示器显示   │ ✅ 完成
└──────────────┘
```

## ❓ 常见问题

### Q1: 如何验证消息已保存到数据库？

**A**: 在 PostgreSQL 中执行：

```sql
-- 查看预录消息
SELECT message_id, message_name, message_content, ins_time
FROM dbo.dmd_pre_record_message
ORDER BY ins_time DESC LIMIT 5;

-- 查看播放列表
SELECT playlist_id, device_id, message_id, send_time
FROM dbo.dmd_playlist
ORDER BY ins_time DESC LIMIT 5;
```

### Q2: MSMQ 发送失败怎么办？

**A**: MSMQ 失败不影响数据库操作，消息已经保存。可以：
1. 启动 DCU Service
2. 手动触发 Task 重新读取数据库
3. 或者重新发送消息（会更新数据库）

### Q3: 显示器没有收到消息？

**A**: 检查以下项目：
1. DCU Service 是否运行
2. 串口是否连接正确
3. 目标设备ID是否匹配
4. 查看 DCU Service 日志

### Q4: 如何自定义更多参数？

**A**: 查看 `消息发送功能_界面设计说明.md`，实现完整版界面（约 40 个控件）

### Q5: 可以同时发送到多个设备吗？

**A**: 当前简化版不支持。需要实现完整版或者多次发送。
