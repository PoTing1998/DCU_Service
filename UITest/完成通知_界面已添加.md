## 📝 使用流程

### 快速测试（3步完成）

1. **点击"填充默认值"**
   - 自动填充测试消息和目标设备

2. **配置数据库连接**（如未配置）
   - 在左侧"资料库连线设定"区域配置
   - 点击"测试连线"验证

3. **点击"发送测试消息"**
   - 查看执行结果
   - 检查显示器是否收到消息

### 详细步骤

#### 1. 配置数据库（首次使用）

在左侧"资料库连线设定"区域：
- 伺服器: PostgreSQL 服务器地址
- 连接埠: 5432（默认）
- 资料库: 数据库名称
- 使用者ID: 用户名
- 密码: 密码
- 点击"更新连线字串"
- 点击"测试连线"验证

#### 2. 填写消息内容

在右侧"消息发送测试"区域：
- **消息内容**: 输入要显示的文字（例如："测试消息123"）
- **目标设备**: 输入设备ID（格式：StationID_AreaID_DeviceID，例如："LG01_CCS_CDU-1"）
- **即时消息**: 勾选框（勾选=即时消息，不勾选=预录消息）

#### 3. 发送消息

点击 **"发送测试消息"** 按钮，查看结果显示：

```
===== 开始发送测试消息 =====
[1/5] 生成消息 ID
      ✓ 消息 ID: ...
[2/5] 解析目标设备
      ✓ 车站: LG01, 区域: CCS, 设备: CDU-1
[3/5] 插入消息到数据库
      ✓ 消息插入成功
[4/5] 更新播放列表
      ✓ 播放列表更新成功
[5/5] 发送 MSMQ 通知
      ✓ MSMQ 通知发送成功
===== 消息发送完成 =====
```

## 📊 功能说明

### 自动使用的默认参数

简化版会自动使用以下参数（用户无需输入）：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 消息类型 | 71 (FullWindow) | 全窗口显示 |
| 优先级 | 1 | 最高优先级 |
| 显示模式 | 100 (静态) | 不滚动 |
| 滚动速度 | 5 | 如果滚动 |
| 字体 | 明體 24pt | 24x24 像素 |
| 颜色 | 白色 | RGB(255,255,255) |

### 数据流程

```
UITest 界面
    ↓
插入到数据库 (dmd_pre_record_message 或 dmd_instant_message)
    ↓
更新播放列表 (dmd_playlist)
    ↓
发送 MSMQ 通知到 TaskCDU/SDU/PUP/PDN
    ↓
DCU Service 接收并从数据库读取
    ↓
组装封包并通过串口发送
    ↓
显示器显示消息 ✅
```

## ⚠️ 注意事项

### 前置条件

1. **PostgreSQL 数据库**
   - 服务必须运行
   - 数据库连接信息正确
   - 相关表已创建（dmd_pre_record_message, dmd_instant_message, dmd_playlist）

2. **DCU Service**（可选，用于 MSMQ 通知）
   - 如果需要实时触发，确保 DCU Service 正在运行
   - 如果 DCU Service 未运行，消息仍会保存到数据库

3. **MSMQ 服务**（可选）
   - Windows 功能中启用 MSMQ
   - 用于触发 DCU Service 处理

### 目标设备格式

必须严格按照格式：`StationID_AreaID_DeviceID`

**正确示例**:
- ✅ LG01_CCS_CDU-1
- ✅ LG01_PLA_PUP-1
- ✅ LG02_CCS_SDU-1

**错误示例**:
- ❌ LG01-CCS-CDU-1 (使用 - 而不是 _)
- ❌ CDU-1 (缺少 Station 和 Area)
- ❌ LG01_CCS_CDU (缺少设备编号)

## 🔧 故障排查

### 问题 1: 编译错误

**错误**: "找不到 Newtonsoft.Json"

**解决**:
```
1. 确认 DLL 路径存在: D:\ASI.Wanda.DMD.DCUService\Newtonsoft.Json.dll
2. 如果路径不存在，修改 UITest.csproj 中的 HintPath
3. 或在 Visual Studio 中: 项目 → 添加引用 → 浏览
```

### 问题 2: 数据库连接失败

**错误**: "could not connect to server"

**检查**:
- PostgreSQL 服务是否运行
- 防火墙设置
- 连接信息是否正确
- 在左侧点击"测试连线"验证

### 问题 3: MSMQ 发送失败

**错误**: "The queue does not exist"

**原因**: DCU Service 未运行或 MSMQ 队列未创建

**解决**:
1. 启动 DCU Service (KernelService)
2. 确认 MSMQ 服务已启用
3. 注意：MSMQ 失败不影响数据库操作

**验证数据库**:
```sql
-- 查看刚插入的消息
SELECT * FROM dbo.dmd_pre_record_message
ORDER BY ins_time DESC LIMIT 5;

-- 查看播放列表
SELECT * FROM dbo.dmd_playlist
WHERE device_id = 'CDU-1'
ORDER BY ins_time DESC LIMIT 5;
```

### 问题 4: 界面控件未显示

**解决**:
1. 重新打开 Form1.cs 设计器
2. 切换到"资料库设定"页面
3. 查看右侧是否有"消息发送测试"区域
4. 如果没有，重新生成项目
